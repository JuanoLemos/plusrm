## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately:

1. **DO NOT** file a public issue on GitHub.
2. Send an email to <security@example.com> with:
   - A description of the vulnerability
   - Steps to reproduce
   - Affected versions
   - Potential impact
3. You should receive a response within 48 hours.
4. If you do not receive a response, follow up via the project's issue tracker
   with a reference to your initial report.

## Disclosure Policy

- Vulnerabilities will be acknowledged within 48 hours.
- A fix will be developed and tested.
- Once a fix is released, a public advisory will be published.
- Reporters will be credited (unless they request anonymity).

## Security Measures

- All dependencies are regularly audited.
- Code reviews are required for all changes.
- Access to production systems is restricted and logged.
